\doxysubsection{Detailed Description}
The main class \doxylink{classpaceval__c_computation}{paceval\+\_\+c\+Computation} encapsulates all the work steps that a user\textquotesingle{}s passed function is analyzed and transferred into the linked list of atomic mathematical operations and stored in memory. 

After successfully completing this analysis, the user can call the \doxylink{classpaceval__c_computation}{paceval\+\_\+c\+Computation} class via paceval\textquotesingle{}s exported API functions with values for the variables and receive results for the calculations.

The do\+Computation function of the \doxylink{classpaceval__c_computation}{paceval\+\_\+c\+Computation} class is the main routine for processing the list of nodes with concrete values for variables. do\+Computation is also used by threads for parallel computation. For this there is the main thread, which runs through the entire list of nodes. The other threads then only calculate a part of the list of nodes in parallel and store the results of the calculation in the lookahead cache. The identification of such parallelization opportunities takes place during the creation of the list of nodes in the \doxylink{classpaceval__c_graph}{paceval\+\_\+c\+Graph} class, see identify\+Multithread\+Opportunity. There is also the option to use inner and outer caching. What is meant by inner chaching is that once a partial function has been calculated, for example "{}sin(5)"{}, it does not have to be calculated again the next time. Outer caching means that if the same calculation is recurring, i.\+e. "{}sin(y)+x\texorpdfstring{$\ast$}{*}sin(y)"{}, then the value is taken from the inner cache for all recurring partial calculations, here "{}sin(y)"{}.
